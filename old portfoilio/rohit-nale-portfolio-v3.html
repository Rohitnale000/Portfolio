Contract-First Interfaces

Data Model Before Feature Growth

Security Boundaries Built In

I tune query plans, connection pools, and background workflows only where load proves pressure. Scaling decisions stay measurable and reversible.

SYS-001 Node.js · TypeScript · GraphQL · Prisma · PostgreSQL

Cordia — Location-Based Dating App

LIVE · Nov 2025
Problem
User discovery required accurate, real-time geo-matching. Traditional SQL distance logic degraded under concurrent traffic and produced inconsistent matches due to Earth-curvature approximations.
System Design
Designed a schema-first GraphQL API backed by Prisma ORM. Geo-distance logic was embedded at resolver level using the Haversine formula for Earth-accurate matching across regions.
Engineering Decisions
Organized resolvers by domain boundaries (auth, discovery, matching) to isolate business logic and maintain scalability. Kept geospatial filtering inside query path instead of post-processing.
Outcome
Shipped full lifecycle onboarding → discovery → filters → matching. Secured GraphQL endpoints with JWT + role-based access, optimized PostgreSQL indexing/query paths, and achieved sub-50ms discovery responses under load.

Panda Stack AI — MicroVM Platform
