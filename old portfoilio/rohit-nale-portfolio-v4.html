01 — About

How I Build Systems

01
Schema Before Code
I design the PostgreSQL schema before writing a single route. A wrong schema at week 1 becomes a migration nightmare at month 6.
02
APIs Are Contracts
Every REST endpoint and GraphQL resolver I ship is a promise. Versioning, validation, and error consistency are not optional — they're the product.
03
Security Is Structural
JWT, RBAC, and input sanitization are built into the architecture from day zero — not patched on after a breach.
04
AI Is Infrastructure
I treat AI integrations like any other external service: rate-limited, retried, typed, and abstracted. Anthropic SDK is part of my production toolkit.
05
Observe Everything
Cron jobs, system monitoring, PM2 cluster management — if I can't see it running, it doesn't count as deployed.

I've spent 3 years in production — not prototypes. The difference is you get paged when things break. That changes how you think about every schema, every query, every auth decision.

My core stack is Node.js + TypeScript on Express.js, with PostgreSQL as primary storage, MongoDB for document-flexible models, and Prisma ORM for type-safe data access. I've shipped real GraphQL APIs, real REST services, and real AI-powered automation pipelines.

I've reverse-engineered production APIs with Burp Suite, built geo-matching algorithms using the Haversine formula, integrated Anthropic AI into live infrastructure, and automated microVM lifecycle management on Fly.io. These aren't side projects — they're billable, production systems.

Currently at Prolance IT. Open to Backend / Full-Stack roles in Pune, Mumbai, Bangalore, Hyderabad or fully Remote.
